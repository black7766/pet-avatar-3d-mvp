# 需求说明

## 背景

我们要做的是“宠物照片生成 3D 动效形象”能力，用于 App 内宠物陪伴、健康反馈、喂食奖励、社交卡片和后续 AI 宠物 Agent。当前 MVP 不是完整 3D skeleton 方案，而是用图生图、视频生成、抠图和透明 WebP/WebM 先跑通可展示链路。

## 已确定能力

上传一张宠物照片后，系统自动生成两版形象：

| 版本 | 目标 |
|---|---|
| 实体版 | 更接近真实宠物，保留毛色、花纹、脸型、体型和识别特征 |
| 萌宠版 | 类似 PaiMomo 风格的 3D 卡通萌宠，适合 App 首页长期陪伴 |

每版当前只支持四种产品状态：

| 状态 | 文件名 | 含义 | 播放方式 |
|---|---|---|---|
| 静息 | idle | 清醒低活动，看向用户，轻微呼吸/眨眼 | 无声循环 |
| 奔跑 | run | 高强度运动或快速玩耍 | 无声循环 |
| 走动 | walk | 低中强度连续移动 | 无声循环 |
| 睡觉 | sleep | 已经入睡后的稳定状态 | 无声循环 |

## 最新产品状态规划

算法和产品状态统一收敛为四大状态：

| 大状态 | 产品含义 | 健康采集 |
|---|---|---|
| 静息 | 清醒但低活动，是绝大多数时间状态 | 采集静息心率、静息呼吸 |
| 睡眠 | 长时间低活动或夜间稳定休息 | 不按普通静息采集，可单独做睡眠监测 |
| 行走 | 低中强度连续移动 | 不做静息心率/呼吸采集 |
| 奔跑 | 高强度移动、冲刺、快速玩耍 | 不做静息心率/呼吸采集 |

静息是基础状态，内部可以维护多个小动作随机播放，但算法分类仍然是静息。猫狗静息动作池来自《猫狗四大状态与静息动作池设计表.xlsx》。

## 交互要求

- 页面首屏直接展示可看的动效成果，不做普通营销落地页。
- 同一只宠物展示原图、实体版、萌宠版。
- 动作切换按钮控制当前展示动作，按钮不能遮挡宠物。
- 每次上传生成都保留在历史里，不覆盖旧结果。
- 历史卡片展示生成耗时、帧数、文件体积、抠图残留等指标。
- 媒体只在页面内预览，不能触发浏览器下载或 IDM 下载。

## 生成要求

- 使用 Seedream 生成两版形象。
- 使用 Seedance 生成动作视频，必须无声，参数里保持 `generate_audio=false`。
- 本地进行绿幕/背景抠图、边缘优化、透明 WebP/WebM 合成。
- 输出需要尽量无缝循环，避免跳帧、闪回初始姿态、变脸和变体型。
- 对任意上传姿态都要统一成标准宠物资产姿态，不能继承原图动作。

## 质量要求

- 保持同一只宠物的身份特征。
- 毛发边缘不能有明显绿边、黑边、碎边。
- 首尾帧差异要小，循环要顺。
- 睡眠状态第一帧就应是已睡着姿态，不做入睡或醒来过渡。
- 走动和奔跑第一帧就应处于对应运动状态，不做起步、停止或回到静息的过渡。

## 当前限制

- 目前不是真正 skeleton + skinning + IK 的实时 3D 模型。
- 当前是 2.5D/视频资产方案，适合先做 App Demo 和投资人展示。
- 真正可交互 3D 需要后续接入 skeleton、动作状态机、蒙皮权重和行为系统。
